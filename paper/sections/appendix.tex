\section{Dataset Details}
\label{app:datasets}

\paragraph{Syn-Logistics Design.}
The Syn-Logistics dataset guarantees $n\geq20$ per subtype via three
parallel 4-hop lineage chains across five components. Table names are drawn
from realistic supply-chain terminology (64 tables, 96 FK, 35 lineage).

\paragraph{Lineage Edge Curation.}
\textbf{AW}: Microsoft reference docs (39 edges).
\textbf{TPC-DS}: No lineage (pure FK).
\textbf{TPC-DI}: Official spec ETL flows (21 edges).
\textbf{OMOP}: OHDSI ETL conventions (21 edges).
\textbf{Syn-Log}: By construction (35 edges).

\section{Evaluation Details}
\label{app:decoding}

All inferences use greedy decoding ($\tau{=}0$, $p{=}1.0$) in a single
deterministic pass. Vector-RAG uses \texttt{all-MiniLM-L6-v2} with
$k{=}15$. List answers are compared as sorted sets after lowercasing.

\section{Example Questions}
\label{app:examples}

\paragraph{Easy (\texttt{direct\_fk}):}
\emph{``Is there a direct FK between \texttt{DW.DimProduct} and
\texttt{DW.FactInternetSales}?''} \textbf{Gold: yes}

\paragraph{Medium (\texttt{join\_path}):}
\emph{``Shortest FK path from \texttt{DW.DimCustomer} to
\texttt{DW.DimProduct}?''} \textbf{Gold: [DimCustomer, FactInternetSales,
DimProduct]}

\paragraph{Hard (\texttt{combined\_impact}):}
\emph{``Which tables are transitively affected if
\texttt{raw\_purchase\_orders} is modified?''}
