\section{Baselines}
\label{sec:baselines}

We implement six baselines with identical question prompts; only the
context injection varies. All use zero-shot prompting.

\textbf{Flat Text (FT)}~serializes the entire schema (tables, columns,
FK edges, lineage) as plain text, simulating a developer loading full
DDL into a long-context LLM.

\textbf{Vector-RAG (VR)}~embeds schema elements with
Sentence-BERT~\cite{reimers2019sentencebert}, retrieves $k{=}15$ chunks
via FAISS, and injects them as context.

\textbf{Graph-Augmented (GA)}~extracts a 3-hop BFS neighborhood from
the schema graph centered on mentioned table names, providing local
topology as structured context. For global questions lacking anchor
tables (e.g., component count), the full graph is provided.

\textbf{Tool-Use (TU)}~provides purpose-built graph tools:
\texttt{get\_neighbors}, \texttt{shortest\_path},
\texttt{get\_connected\_component}, \texttt{get\_lineage\_chain},
and \texttt{list\_tables}. Up to 3 sequential tool calls per question.

\textbf{ReAct-Code (RC)}~generates and executes Python/NetworkX code in
a sandboxed REPL with up to 5 execution rounds, serving as an implicit
budget ablation against TU's 3-call limit. RC
directly on the schema graph, providing maximum algorithmic flexibility.

\textbf{Oracle}~injects gold algorithmic output into the prompt,
establishing the upper bound.

For \textbf{Tier~2}, we adapt three baselines: FT$_{v2}$ (DDL-only),
GA$_{v2}$ (schema + value samples), TU$_{v2}$ (interactive data tools).
