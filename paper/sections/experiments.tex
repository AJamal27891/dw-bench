\section{Experiments}

\subsection{Setup}

We evaluate using two frontier-class models: \textbf{Gemini~2.5 Flash}
(Google, closed-weight) and \textbf{DeepSeek-V3} (DeepSeek, open-weight,
MIT licensed, 671B MoE). Each question receives baseline-specific context
and returns a JSON response with reasoning and answer. We report:

\begin{itemize}
  \item \textbf{Micro-EM}: Overall exact match across all questions
    (weighted by question count).
  \item \textbf{Macro-EM}: Mean of per-subtype exact match rates,
    weighting all 13~subtypes equally regardless of sample size.
  \item \textbf{F1 Score}: Token-level overlap for partial credit on
    list answers.
\end{itemize}

We include an \textbf{Oracle} baseline that injects gold graph algorithm
output into the prompt, simulating perfect retrieval. Oracle scores
establish the upper-bound achievable by each model.

\paragraph{Key Finding~1: The Triviality Illusion.}
Under micro-EM, Flat Text appears competitive (73.9\% for Gemini).
However, switching to macro-EM---which weights all 13~subtypes
equally---reveals that \textbf{Graph-Augmented consistently ranks first}
across both models and both conditions (Figure~\ref{fig:triviality},
Table~\ref{tab:main-results}).
The gap is +6.9\% for Gemini and +17.8\% for DeepSeek. This ``triviality
illusion'' is caused by \texttt{join\_path}'s 316~questions (47\% of all
data) inflating micro-EM for text-only baselines that can handle FK
adjacency but fail on deeper structural tasks.

\paragraph{Key Finding~2: DeepSeek benefits more from graph context.}
DeepSeek-V3 shows a larger GA--FT macro-EM gap (+17.8\%) compared to
Gemini (+6.9\%), suggesting that models with less exposure to
domain-specific schemas in pretraining benefit disproportionately from
explicit graph context injection. This motivates graph-augmented retrieval
as a model-agnostic enhancement.

\paragraph{Key Finding~3: Hard task collapse is model-independent.}
Both Gemini and DeepSeek collapse on hard questions---14.8\% and 31.6\%
for Flat Text respectively---while the Oracle achieves 97.9--98.9\% on
the same questions. This proves the failure is \emph{not} due to
question ambiguity but to retrieval/reasoning inability. The 97.9\%
Oracle EM on hard questions establishes that DW-Bench questions are
computationally solvable when given correct graph context.

\subsection{Per-Subtype Analysis}

\begin{figure}[t]
\centering
\includegraphics[width=0.85\linewidth]{figures/subtype_heatmap.pdf}
\caption{Per-subtype EM heatmap (Gemini). Subtypes sorted by Graph-Aug
performance. Bottom rows (\texttt{combined\_impact}, \texttt{membership})
remain universally hard.}
\label{fig:heatmap}
\end{figure}

\paragraph{Key Finding~4: Specific subtypes reveal graph reasoning gaps.}
\texttt{membership} (0\% EM for DeepSeek FT/VR), \texttt{transitive}
(0\% EM for DeepSeek FT/VR), and \texttt{combined\_impact}
($\leq$6.5\% EM across all baselines) represent fundamentally unsolved
structural reasoning tasks. Only Graph-Augmented achieves non-zero
scores on these subtypes, confirming that explicit graph context is
necessary but insufficient for deep topology reasoning.

\paragraph{Key Finding~5: Combined impact remains unsolved.}
The \texttt{combined\_impact} subtype---requiring simultaneous traversal
of lineage chains AND FK dependency trees---achieves $\leq$6.5\% EM across
all baselines on Gemini and $\leq$14.3\% on DeepSeek. This establishes a
clear frontier for future work on learned graph representations.

\paragraph{Key Finding~6: Obfuscation reveals memorization asymmetry.}
DeepSeek suffers dramatically larger obfuscation penalties ($-$27.7\%
to $-$30.1\%) compared to Gemini ($-$9.0\% to $-$21.3\%). This suggests
Gemini has greater pretraining exposure to these Western enterprise
schemas (AdventureWorks, TPC benchmarks), enabling partial memorization
that survives in original evaluations. When this crutch is removed via
obfuscation, DeepSeek's true structural reasoning ability is revealed.
Notably, Graph-Augmented has the \emph{smallest relative obfuscation
penalty} on DeepSeek ($-$23.1\% vs.\ $-$30.1\% for VR), confirming
that graph structure partially compensates for lost name semantics.
